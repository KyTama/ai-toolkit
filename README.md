# ai-toolkit

Personal AI dev environment — unified GEMINI.md + CLAUDE.md management,
antigravity-kit agents/workflows, and curated skills.

---

## First Time Setup
```bash
git clone https://github.com/KyTama/ai-toolkit.git
cd ai-toolkit
```

Copy and fill in your personal context:
```bash
cp _personal.md.example _personal.md
vi _personal.md
```

Then bootstrap:
```bash
./scripts/bootstrap.sh
```

Then add aliases to your shell:
```bash
echo 'alias ai-bootstrap="~/projects/ai-toolkit/scripts/bootstrap.sh"' >> ~/.zshrc
echo 'alias ai-rebuild="~/projects/ai-toolkit/scripts/rebuild-global.sh"' >> ~/.zshrc
echo 'alias ai-update="~/projects/ai-toolkit/scripts/update.sh"' >> ~/.zshrc
source ~/.zshrc
```

> Adjust `~/projects/ai-toolkit` to wherever you cloned this repo.

## Setup a New or Existing Project

### New project
```bash
cd ~/projects/your-project
ai-bootstrap .
```

### Existing project (no GEMINI.md / CLAUDE.md yet)

Same as above — bootstrap detects no existing files and creates them safely.

### Existing project (already has GEMINI.md / CLAUDE.md)

Bootstrap will **skip** existing files by default to avoid overwriting your context:
```
→ GEMINI.md exists, skipping (use --force to override)
```

If you want to **replace** the existing files with the ai-toolkit template (your
old files will be backed up as `GEMINI.md.bak`):
```bash
ai-bootstrap . --force
```

If you want to **merge** your existing context with the ai-toolkit template
(recommended when you have custom project context you want to keep):
```bash
# Backup first
cp GEMINI.md GEMINI.md.bak

# Generate fresh template
cp ~/projects/ai-toolkit/templates/GEMINI.project.md GEMINI.md

# Manually copy your Project Context section from the backup into the new file
vi GEMINI.md
```

Then edit the Project Context section at the top:
```
## Project Name
## Stack
## Current Phase
## Active Tasks
```

## Where to Add Context

Use this as a guide for deciding where to put new context:

| What | Where | Tracked |
|---|---|---|
| Tone, communication style, role | `_shared.md` | yes |
| Code style rules | `_shared.md` | yes |
| Workflow philosophy | `_shared.md` | yes |
| Your tech stack | `_personal.md` | no |
| Your active projects | `_personal.md` | no |
| Anything personal/private | `_personal.md` | no |
| Gemini CLI / Antigravity behavior | `_gemini_specific.md` | yes |
| Orchestration and planning rules | `_gemini_specific.md` | yes |
| Claude Code behavior | `_claude_specific.md` | yes |
| Current project details | `GEMINI.md` / `CLAUDE.md` in project root | depends |

**Rule of thumb:**
- Adding context about yourself → `_shared.md` or `_personal.md`
- Adding behavior rules specific to one tool → `_gemini_specific.md` or `_claude_specific.md`
- Adding context about the current project → edit `GEMINI.md` in the project root

You **never need to touch** `_gemini_specific.md` or `_claude_specific.md` for
personal context — those files are for tool-specific behavior only.

### When would you manually edit `_gemini_specific.md` or `_claude_specific.md`?

Only when you want to change **how the AI behaves**, not what it knows about you.
Real examples:

- You want Gemini to always run tests before marking a task done → `_gemini_specific.md`
- You want Claude to always use a specific output format → `_claude_specific.md`
- You discovered a new orchestration pattern that works well → `_gemini_specific.md`
- You want Claude to prefer a specific MCP tool → `_claude_specific.md`
- You want to add a new slash command convention → `_gemini_specific.md`

After any edit to source files, always run:
```bash
ai-rebuild
```

## Important: Assembled Files

`GEMINI.md` and `CLAUDE.md` (both global and per-project) are **assembled files**
— they are generated by concatenating source files. They are not live-linked.
```
_shared.md
_personal.md          →  ai-rebuild  →  ~/.gemini/GEMINI.md
_gemini_specific.md                     ~/.claude/CLAUDE.md
```

**Never edit the assembled files directly** — your changes will be overwritten
the next time `ai-rebuild` or `ai-bootstrap` is run.

The only exception is the **Project Context section** at the top of each
per-project `GEMINI.md` / `CLAUDE.md` — that section is yours to edit freely
and won't be touched unless you run `ai-bootstrap . --force`.

## Daily Usage

### Triggering skills and workflows

Skills are loaded on-demand based on context. Trigger them via slash commands:
```
/plan build authentication system
/debug why is this endpoint returning 500
/brainstorm architecture for reservation system
/enhance refactor this service layer
/test generate tests for this module
```

Or reference a skill directly:
```
use systematic-debugging to trace this bug
use bun-development for this task
use docker-expert to review this Dockerfile
```

## Alias Cheatsheet

| Command | Description |
|---|---|
| `ai-bootstrap .` | Bootstrap current directory as a project |
| `ai-bootstrap . --force` | Force overwrite existing GEMINI.md / CLAUDE.md (backs up old files) |
| `ai-bootstrap ~/projects/x` | Bootstrap a specific project path |
| `ai-rebuild` | Rebuild global GEMINI.md + CLAUDE.md |
| `ai-update` | Pull latest from antigravity-kit + awesome-skills |

## How It Works

### Global config (auto-loaded by all tools)
| File | Loaded by |
|---|---|
| `~/.gemini/GEMINI.md` | Gemini CLI + Antigravity (global fallback) |
| `~/.claude/CLAUDE.md` | Claude Code (global fallback) |

### Per-project config (auto-loaded when project is opened)
| File/Folder | Purpose |
|---|---|
| `GEMINI.md` | Project context + shared identity + Gemini behavior |
| `CLAUDE.md` | Project context + shared identity + Claude behavior |
| `.agent/agents/` | 20 specialist AI personas |
| `.agent/skills/` | 200+ domain skill modules |
| `.agent/workflows/` | Slash command procedures |

### Source of truth
| File | Tracked | Purpose |
|---|---|---|
| `_shared.md` | yes | Identity, tone, code style, workflow philosophy |
| `_personal.md` | no | Stack, active projects (local only) |
| `_gemini_specific.md` | yes | Gemini CLI + Antigravity behavior + orchestration rules |
| `_claude_specific.md` | yes | Claude Code behavior overrides |

## Structure
```
ai-toolkit/
├── .agent/
│   ├── agents/       # specialist AI personas (from antigravity-kit)
│   ├── skills/       # domain skill modules (ag-kit + awesome-skills)
│   └── workflows/    # slash command procedures
├── scripts/
│   ├── bootstrap.sh       # setup global config + project
│   ├── rebuild-global.sh  # rebuild global config only
│   └── update.sh          # pull latest from upstream
├── templates/
│   ├── GEMINI.project.md  # per-project GEMINI.md template
│   └── CLAUDE.project.md  # per-project CLAUDE.md template
├── _shared.md             # identity, tone, code style (tracked)
├── _personal.md           # stack, projects (local only, gitignored)
├── _personal.md.example   # template for _personal.md (tracked)
├── _gemini_specific.md    # gemini/antigravity overrides (tracked)
└── _claude_specific.md    # claude code overrides (tracked)
```
